# Учебная и производственная в Neoflex. Лето 2022

## Дудко Матвей

### Тема

Разработка RESTful API сервиса для создания расписания онлайн-школы.

### Актуальность

Тренд последнего десятилетия - онлайн образование. Все больше и больше преподавателей начинают преподавать на огромном количестве онлайн курсов.

### Цели и задачи

Для структурирования данных необходимо спроектировать и реализовать инструмент для создания курса, составления расписания и трекинга активности преподавателей.

### План и структурирование работы

1. Спроектировать модель данных:

    1. Курс:

       У курса обязательно должны быть:

        - название;
        - описание;
            - категория (например технические, которые включают в себя языки программирования, технологии и инструменты). Необходимо сделать иерархию категорий на основе наследования;
            - куратор (ссылка на преподавателя - куратора курса);
            - количество студентов;
            - список занятий.
        - тип (например онлайн или оффлайн, у каждого свой набор полей, у онлайна может быть ссылка на платформу проведения, у оффлайн название вуза и адрес).

    2. Преподаватель:
        - ФИО;
        - возраст.

    3. Занятие:
        - название;
        - описание;
        - преподаватель;
        - дата и время проведения.


2. Реализовать API для управление моделью (CRUD), необходимые валидации и исключения. Необходимо иметь возможность создать курс, добавить в него занятия, назначить/поменять на каждое занятие преподавателя.


3. Имплементировать ролевую модель. Каждый преподаватель получает роль ROLE_TEACHER при регистрации. Он может создавать курсы и занятия, редактировать и добавлять занятия ТОЛЬКО к своим курсам. Добавить пользователя с ролью ROLE_ADMIN, который может редактировать все курсы/занятия.


4. Реализовать RESTful API для:
    - создания курса копированием другого курса, при этом куратором назначается тот, кто копирует курс. Список занятий тоже копируется, но дата и время зануляются;
    - получение расписания каждого преподавателя.


5. Задокументировать API при помощи Swagger.
